# บทที่ 6: Terminal Value

## Terminal Value คืออะไร?

**มูลค่าสุดท้าย** ของบริษัทหลังจากช่วงที่คาดการณ์ได้

> มักคิดเป็น **60-80%** ของ Valuation ทั้งหมด!

## 2 วิธีคำนวณ

### 1. Gordon Growth Model (Perpetuity)

สมมติบริษัท **เติบโตตลอดไป** ด้วยอัตราคงที่

```
Terminal Value = FCFF_(n+1) / (WACC - g)
```

- `FCFF_(n+1)` = Free Cash Flow ปีถัดจากช่วงคาดการณ์
- `g` = Growth Rate ถาวร (ต้อง < WACC)

**Constraint:** `g` ต้องไม่เกิน Growth ของเศรษฐกิจ (~2-3%)

### 2. Exit Multiple

ใช้ **คูณ** กับตัวเลขทางการเงิน

```
Terminal Value = EBITDA × EV/EBITDA Multiple
```

หรือ

```
Terminal Value = Earnings × P/E Multiple
```

## เปรียบเทียบ 2 วิธี

| | Gordon Growth | Exit Multiple |
|---|--------------|---------------|
| พื้นฐาน | Fundamentals | Market-based |
| ข้อดี | Consistent กับ DCF | ง่าย, เข้าใจง่าย |
| ข้อเสีย | Sensitive ต่อ g | Multiple อาจผิด |

## Damodaran แนะนำ

ใช้ **Gordon Growth** เป็นหลัก เพราะ:
- Consistent กับ DCF philosophy
- ตรวจสอบได้ด้วย Exit Multiple

## การตั้งค่า Growth Rate (g)

```
g ≤ Risk-Free Rate ≈ Growth ของเศรษฐกิจ
```

### ตัวอย่าง

- Risk-Free Rate (US) = 4%
- ฉะนั้น `g` ไม่ควรเกิน 3-4%

## ความเสี่ยงของ Terminal Value

1. **Sensitivity สูง** - เปลี่ยน g นิดเดียว มูลค่าเปลี่ยนมาก
2. **Assumptions ยาวนาน** - สมมติไป 10+ ปี
3. **Ignore competition**

## วิธีตรวจสอบ

1. **Check Implied Multiple** - แปลงกลับเป็น P/E, EV/EBITDA
2. **Compare with Industry** - ดูว่าสมเหตุสมผลไหม
3. **Stress Test** - ลองเปลี่ยน assumptions
