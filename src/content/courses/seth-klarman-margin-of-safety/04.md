# บทที่ 4: Scenario Analysis

## Scenario Analysis คืออะไร?

**Scenario Analysis** = วิเคราะห์หลายสถานการณ์ แทนการคาดการณ์แค่สถานการณ์เดียว

> "We don't know what will happen, but we can prepare for multiple outcomes."
> — Seth Klarman

เราไม่รู้อนาคต แต่เราสามารถคำนวณว่า "ถ้าเกิดแบบนี้ จะเกิดอะไร"

## ทำไมต้องใช้ Scenario Analysis?

### ปัญหาของการคาดการณ์แบบเดียว

```
❌ Base Case: เติบโต 8% ต่อปี → ราคาเป้า 100 บาท
```

ถ้า assumption ผิด → ตัวเลขผิดทั้งหมด

### แนวทาง Seth Klarman

```
✅ หลาย Scenarios → ดูความเป็นไปได้ทั้งหมด
✅ Weight by Probability → คำนวณ Expected Value
✅ Focus on Downside → ถ้า worst case รับได้ → ค่อยลงทุน
```

## 3 Scenarios พื้นฐาน

| Scenario | คำอธิบาย | Probability |
|---|---|---|
| **Bull Case** | ทุกอย่างดีเกินคาด | 15-25% |
| **Base Case** | เป็นไปตามคาดการณ์ | 50-60% |
| **Bear Case** | ทุกอย่างแย่กว่าคาด | 20-30% |

> ผลรวม Probability ต้อง = 100%

## ขยายเป็น 5 Scenarios (แบบ Klarman)

| Scenario | คำอธิบาย | Probability |
|---|---|---|
| **Very Bull** | ดีมาก (M&A, catalyst ใหญ่) | 5-10% |
| **Bull** | ดีกว่าคาด | 15-20% |
| **Base** | ปกติ | 40-50% |
| **Bear** | แย่กว่าคาด | 15-25% |
| **Disaster** | พังทลาย | 5-10% |

## วิธีคำนวณ Expected Value

### สูตร

```
Expected Value = Σ (Probability_i × Return_i)

ถ้า EV > 0 → น่าสนใจ
ถ้า EV < 0 → ไม่คุ้ม
```

### ตัวอย่าง

| Scenario | Probability | Return | Contribution |
|---|---|---|---|
| Very Bull | 10% | +80% | +8% |
| Bull | 20% | +40% | +8% |
| Base | 45% | +15% | +6.75% |
| Bear | 20% | -20% | -4% |
| Disaster | 5% | -50% | -2.5% |
| **Total** | **100%** | | **+16.25%** |

**Expected Value = +16.25%** ✅ น่าสนใจ

## ตัวอย่าง: ADVANC (สมมติ)

> **Context:** ADVANC ราคา 200 บาท กำลังจะประมูล 5G

### Scenario 1: ประมูลได้ + ราคาดี

| รายละเอียด | ค่า |
|---|---|
| Probability | 15% |
| ราคาเป้า | 280 บาท |
| Return | +40% |
| Contribution | +6% |

### Scenario 2: ประมูลได้ + ราคาปานกลาง

| รายละเอียด | ค่า |
|---|---|
| Probability | 30% |
| ราคาเป้า | 240 บาท |
| Return | +20% |
| Contribution | +6% |

### Scenario 3: ไม่ได้ประมูล + ยังดี

| รายละเอียด | ค่า |
|---|---|
| Probability | 30% |
| ราคาเป้า | 210 บาท |
| Return | +5% |
| Contribution | +1.5% |

### Scenario 4: ประมูลได้แพง + กดดัน margin

| รายละเอียด | ค่า |
|---|---|
| Probability | 15% |
| ราคาเป้า | 170 บาท |
| Return | -15% |
| Contribution | -2.25% |

### Scenario 5: Disaster (เกิด price war รุนแรง)

| รายละเอียด | ค่า |
|---|---|
| Probability | 10% |
| ราคาเป้า | 130 บาท |
| Return | -35% |
| Contribution | -3.5% |

### สรุป

```
Expected Value = 6% + 6% + 1.5% - 2.25% - 3.5%
               = +7.75%

Worst Case: -35% (ถ้า Disaster)
Best Case: +40% (ถ้า Very Bull)
```

**สรุป:** EV บวก แต่ต้องยอมรับ Downside -35% ได้

## สิ่งที่ต้องใส่ใจ

### 1. Probability เป็น Subjective

เรากำหนดเอง → อาจผิดได้

```
❌ คิดว่า Disaster โอกาส 5% → จริงๆ อาจ 15%
✅ ใช้ probability แบบ conservative
```

### 2. Correlation ระหว่าง Scenarios

บางครั้ง scenarios ไม่ independent

```
ถ้าเศรษฐกิจถดถอย → หลายหุ้นจะลงพร้อมกัน
```

### 3. Fat Tail Risk

ตลาดการเงินมี "fat tail" — extreme events เกิดบ่อยกว่าที่คิด

| Event | คาดการณ์ (Normal) | จริง |
|---|---|---|
| -3 SD move | 0.13% | ~1-2% |
| -4 SD move | 0.003% | ~0.5% |

> ถ้าคิดว่า "ไม่มีทางเกิด" → มันจะเกิด

## Scenario Analysis vs Sensitivity Analysis

| วิธี | ทำอะไร | ใช้เมื่อ |
|---|---|---|
| **Scenario** | เปลี่ยนหลาย variables พร้อมกัน | ดู big picture |
| **Sensitivity** | เปลี่ยนทีละ variable | ดูว่าตัวไหนสำคัญ |

**Sensitivity ตัวอย่าง:**

| Variable | เปลี่ยน | ผลต่อ Valuation |
|---|---|---|
| Growth Rate | 5% → 3% | -15% |
| Discount Rate | 10% → 12% | -20% |
| Margin | 20% → 15% | -25% |

→ Margin สำคัญที่สุด (ต้องวิเคราะห์ละเอียด)

## กับดักที่ต้องระวัง

### 1. False Precision
คิดว่า probability แม่นยำ

### 2. Ignoring Correlation
ลืมว่า scenarios มีผลต่อกัน

### 3. Overweighting Base Case
ให้น้ำหนัก Base Case มากเกินไป

### 4. Underweighting Tail Risk
ลืมว่า extreme events เกิดได้

## Mindset ของ Seth Klarman

| Do | Don't |
|---|---|
| วิเคราะห์อย่างน้อย 3 scenarios | ใช้แค่ base case |
| ให้ weight กับ downside scenarios | มองข้าม worst case |
| ยอมรับว่า probability เป็น estimation | คิดว่าแม่นยำ |
| ถ้า worst case รับไม่ได้ → ไม่ซื้อ | หวังว่าจะไม่เกิด |

> "It's better to be approximately right than precisely wrong."
> — Seth Klarman

## สรุปบทที่ 4

- **Scenario Analysis** = วิเคราะห์หลายสถานการณ์ แทนแค่สถานการณ์เดียว
- อย่างน้อย 3 scenarios: Bull, Base, Bear
- คำนวณ **Expected Value** = Σ (Probability × Return)
- ให้น้ำหนักกับ **Downside Scenarios** มากขึ้น
- ถ้า Worst Case รับไม่ได้ → **ไม่ลงทุน** แม้ EV บวก
- Fat Tail Risk — extreme events เกิดบ่อยกว่าที่คิด
